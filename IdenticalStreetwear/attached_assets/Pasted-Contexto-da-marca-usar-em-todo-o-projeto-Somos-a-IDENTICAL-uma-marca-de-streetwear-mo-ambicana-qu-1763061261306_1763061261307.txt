Contexto da marca — usar em todo o projeto
Somos a IDENTICAL, uma marca de streetwear moçambicana que valoriza a cultura urbana, autenticidade e influência da arte, música e estilo de vida das ruas. Nosso objetivo: vender coleções / t-shirts “raízes urbanas” com forte identidade local e UX simples para cliente comprar em poucos cliques.
Objetivo do projeto
Desenvolver uma loja web responsiva hospedada no Replit, seguindo estritamente a estrutura, navegação, fluxo e formas de pagamento do site-exemplo https://lerramz.com/. A aparência visual deve ser inspirada no conteúdo e layout do site-exemplo, mas com identidade visual da IDENTICAL (eu fornecerei imagens e logos).

Requisitos técnicos e stack sugerido


Frontend: React (Vite) + Tailwind CSS (ou plain CSS se preferir). Fornecer componentes reutilizáveis (Header, Footer, ProductCard, CartDrawer, CheckoutForm).


Backend: Node.js + Express (API REST) com rotas:


POST /api/auth/register, POST /api/auth/login


GET /api/products, GET /api/products/:id


POST /api/cart, GET /api/cart, POST /api/checkout (inicia o pagamento via PaySuite)


POST /api/paysuite/webhook (recebe notificações)


GET /api/admin/orders, PUT /api/admin/orders/:id/status




BD: SQLite (fácil no Replit) ou Firestore (se quiser cloud). Recomendo SQLite para protótipo e possibilidade de migrar depois.


Armazenamento de imagens: Firebase Storage ou Replit file system (pasta /public/assets/ para imagens fornecidas).


Autenticação: JWT + cookies ou sessions via cookie (secure). Proteger rotas admin.


Env vars: PAYSUITE_API_KEY, PAYSUITE_SECRET, SMTP credentials, JWT_SECRET.



Integração PaySuite — instruções práticas (inclusas no código)


Use a API de criação de pagamento que retorna checkout_url para redirecionar o cliente. Exemplo do fluxo:


Cliente confirma pedido → POST /api/checkout → servidor cria pagamento via PaySuite API com amount, reference, return_url, callback_url.


PaySuite retorna checkout_url → redirecionar cliente para esse URL (ou abrir em iframe/novo tab se necessário).


PaySuite envia webhook para callback_url configurado → servidor atualiza status do pedido. Validar status e signature conforme docs. docs.paysuite.co.mz+1




Incluir exemplo de SDK (PHP/Node) ou uso directo com fetch/axios. Mostrar como testar em sandbox.


Incluir documentação de como instalar PaySuite no painel merchant (se usar gateway alternativo) e como gerar Payment Links. docs.paysuite.co.mz+1
